使用 jsDelivr 加速 GitHub Pages

2020.08.14
#网站

jsDelivr是什么呢，说白了啊，这个东西其实就是一个开源的CDN项目，他会把npm、Github、wordpress仓库中的静态文件分发到全球的CDN节点上，通过访问离我们最近的CDN节点即可访问我们存储在github上的静态文件，加快了访问的速度也节省了时间。
具体如何操作呢，其实你存储在github平台上的静态资源已经自动的分发至cdn节点了，你只需要从jsDelivr上直接调用他们就可以。

我们的网站访问之所以会比较慢，其实就是我们的静态文件加载不及时所导致的，比如css、js等文件，这类文件一般比较大，像如果github服务器在国外的话，加载起来就更慢了。我们要做的就是把静态网页渲染文件中的这些静态资源的链接改为jsDelivr的cdn链接的就好啦,让它在加载过程中直接调用国内的cdn资源

操作：

如： 原链接：https://github.com/Hardisonxxx/hardisonxxx.github.io/styles/main.css
改为https://cdn.jsdelivr.net/gh/Hardisonxxx/hardisonxxx.github.io/styles/main.css
具体来说就是将 "https://github.com/"改为“https://cdn.jsdelivr.net/gh/”
